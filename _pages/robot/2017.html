2017 Robot

This year's challenge is called FIRST STEAMworks. It is steampunk themed, and is focused around moving balls and gears between locations to score points. For a more detailed description, see the official video reveal.

The build season is currently going on, and our robot is still a work in progress. So far we have a shooter, gear mechanism and climber.

Robot Abilities
  • Swerve Drive: This year's game relies heavily on agility. The swerve drive train gives us a huge advantage while picking up and placing gears.
  • Ball collection/shooting apparatus: Our robot has the capacity to rapidly pick up, store and shoot balls. We have an intake on the underside of our robot which allows us to pick up balls by driving into them. We can hold 15 balls at any time. Our shooter has been able to achieve 90% accuracy at 3 balls per second. We're still working on this and it may be modified before the final robot is released.
  • Gear arm: Our robot can deploy a gear collection arm to pick gears up off the floor. This means we do not have to rely on delivery from dispensers, which may be unreliable.
  • Rope climb: Our robot can climb the rope in 12 seconds currently, using a winch mechanism.
Programming

Just like in previous years, we used Python for our robot code. The code uses “MagicRobot”, a part of the RobotPy library that makes using objects between classes easy. It is object-oriented, and controls our arm using custom PID control.

In addition to our robot code, we developed a dashboard for our touchscreen driver station. It is written in JavaScript/HTML/CSS, and communicates with the robot through a python-based tornado server. It is based on the FRCDashboard framework we developed last year.
